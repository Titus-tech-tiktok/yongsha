# 财神测款机 V8 计费系统开发说明

## 1. 目标与当前实现

计费系统与团队账号绑定，按账号对应的独立 `workspaceId` 记录余额和费用。管理员可以：

- 开启或关闭计费。
- 设置“成功生图 / 张”和“语言模型 / 次”的单价。
- 设置新账号首次建立计费账户时的初始余额。
- 给任意团队账号充值或扣减余额。
- 查看全团队最近费用流水。

普通成员可以：

- 在左侧底部看到自己的余额。
- 点击余额查看自己的单价、可用余额和最近费用流水。
- 余额不足时在调用上游模型前收到明确提示，不会继续消耗 API。

系统初始状态为“计费关闭、单价 0、初始余额 0”，因此升级后不会突然阻断现有任务。管理员配置价格、充值并开启计费后才正式生效。

## 2. 模型调用审计与计费范围

### 2.1 图片生成：成功返回一张结果计费一次

| 业务入口 | 实际接口 | 计费说明 |
| --- | --- | --- |
| 素材生图：套图换印花 | `POST /v1/images/edits` | 每张成功生成的套图结果扣一次生图费 |
| 人工筛图：单张重新生成、整套重新生成 | `POST /v1/images/edits` | 每张成功重新生成的结果扣一次生图费 |
| 旧母版模式：母版图生成 | `POST /v1/images/edits` | 成功生成母版图扣一次生图费 |
| 自由生图 | `POST /v1/images/edits` | 成功返回结果扣一次生图费 |

图片接口内部遇到 CPU 繁忙、网络波动时可能自动重试。一次业务生成任务只建立一个计费预占，重试成功后只扣一次；全部失败则释放预占，不扣费。

“直接复制模板”“保留原图”“不处理”等不调用图片模型的动作不计费。

### 2.2 语言模型：每次成功接口调用计费一次

| 业务入口 | 实际接口 | 计费说明 |
| --- | --- | --- |
| 套图模板 AI 分析 | `POST /v1/chat/completions` | 每张模板分析成功扣一次 LLM 费 |
| 商品图片理解 / 商品资料分析 | `POST /v1/chat/completions` | 分析成功扣一次 LLM 费 |
| 生成结果 AI 质检 | `POST /v1/chat/completions` | 首次质检成功扣一次 LLM 费 |
| 生成结果 AI 复核 | `POST /v1/chat/completions` | 首次质检不通过后触发复核；复核成功再扣一次 LLM 费 |

模板分析的失败自动重试只对最终成功的那次计费；失败调用会释放预占。

### 2.3 明确不计费的功能

以下操作不产生业务费用：

- `GET /v1/models` 获取模型列表。
- API 设置里的“测试模型”和“测试分析”连接测试。
- 本地标题生成。
- 素材上传、扫描、缩略图、删除、文件复制。
- 人工筛图中的通过、不通过、打开文件夹等本地操作。
- 计费规则关闭时的所有模型调用。

## 3. 扣费规则

### 3.1 金额精度

所有金额在服务端使用整数“分”保存，例如：

- `$1.000000` 保存为 `1000000`。
- `$0.002362` 保存为 `2362`。

前端只负责将美元金额转换为 6 位小数单位，服务端再次验证必须是安全整数，避免浮点累计误差。

### 3.2 预占、成功、失败

为防止多个并发任务同时花超余额，模型调用分三步：

1. **预占**：调用上游 API 前检查可用余额并预占本次预计费用。
2. **成功提交**：成功获得图片或成功收到 LLM 响应后，正式扣减余额并写费用流水。
3. **失败释放**：上游失败、超时或重试全部失败时，释放预占，不写扣费流水。

可用余额计算公式：

```text
可用余额 = 账户余额 - 正在执行任务的预占金额
```

预占记录持久化在计费账户文件中。服务异常中断后，超过 2 小时的无效预占会自动清理，不会永久占用余额。

### 3.3 价格变更

任务开始预占时固定使用当时单价。管理员保存新价格后：

- 已经开始的调用继续按原预占价格结算。
- 之后新发起的调用使用新价格。

### 3.4 初始余额

“新账号初始余额”只在该账号第一次建立计费账户时发放一次。修改该规则不会给已有账号重复充值；已有账号余额需要管理员单独调整。

## 4. 权限

| 功能 | 管理员 | 普通成员 |
| --- | --- | --- |
| 查看自己的余额和流水 | 支持 | 支持 |
| 查看计费管理页 | 支持 | 不显示且接口拒绝 |
| 修改计费规则 | 支持 | 接口返回 403 |
| 查看全团队余额 | 支持 | 接口返回 403 |
| 调整团队余额 | 支持 | 接口返回 403 |

权限判断在服务端执行，不依赖前端隐藏按钮。

## 5. 服务端数据文件

全部计费数据位于服务器 `CAISHEN_DATA_DIR/system`：

```text
data/system/
├── billing-rules.json       # 全局计费规则
├── billing-accounts.json    # 各 workspace 余额与执行中预占
└── billing-ledger.jsonl     # 追加写入的费用流水
```

团队账号仍位于：

```text
data/system/users.json
```

计费账户通过 `workspaceId` 与团队账号关联，不使用用户电脑的本地路径。

### 5.1 规则文件示例

```json
{
  "version": 1,
  "enabled": true,
  "currency": "USD",
  "amountScale": 1000000,
  "imageFeeMinMinor": 290000,
  "imageFeeMaxMinor": 390000,
  "llmFeeMinMinor": 10000,
  "llmFeeMaxMinor": 30000,
  "defaultBalanceMinor": 100000000,
  "updatedAt": "2026-07-12T00:00:00.000Z"
}
```

上述示例表示：成功生图 `$1.000000/张`，成功 LLM 调用 `$0.002362/次`，新账号初始余额 `$100.000000`。

## 6. 接口

所有接口都要求先登录。

### 6.1 当前账号费用

```http
GET /api/billing/me
```

返回当前账号计费规则、余额、可用余额、预占金额和最近 30 条流水。

### 6.2 管理员计费总览

```http
GET /api/billing/admin
```

仅管理员。返回计费规则、团队账号余额和最近 150 条全局流水。

### 6.3 保存计费规则

```http
PUT /api/billing/rules
Content-Type: application/json

{
  "enabled": true,
  "imageFeeMinMinor": 290000,
  "imageFeeMaxMinor": 390000,
  "llmFeeMinMinor": 10000,
  "llmFeeMaxMinor": 30000,
  "defaultBalanceMinor": 100000000
}
```

### 6.4 调整余额

```http
POST /api/billing/adjust
Content-Type: application/json

{
  "userId": "团队用户 ID",
  "amountMinor": 10000,
  "description": "管理员充值"
}
```

- 正数充值。
- 负数扣减。
- 不允许扣成负余额。
- 流水会记录操作管理员 ID。

## 7. 前端交互

### 7.1 左侧余额

左侧导航底部固定显示：

- 当前账户余额。
- 当前生图单价和分析单价；计费关闭时显示“计费暂未启用”。
- 点击后打开个人费用明细弹窗。

后台任务完成或失败后，页面会自动刷新余额。用户也可以在明细弹窗手动刷新。

### 7.2 管理员计费管理

管理员进入“系统设置 → 计费管理”后可以：

- 启用/关闭计费。
- 设置两种单价和初始余额。
- 查看每个账号余额。
- 输入正数或负数调整余额。
- 查看全团队最近费用流水。

普通成员不会看到“计费管理”标签页。

## 9. 部署、备份和恢复

### 9.1 部署

计费系统不需要额外数据库和新依赖。部署新代码后按项目原方式执行：

```bash
npm ci
npm run check
npm start
```

服务器进程必须对 `CAISHEN_DATA_DIR/system` 有读写权限。

### 9.2 备份

至少备份以下整个目录：

```text
$CAISHEN_DATA_DIR/system/
```

建议与 `workspaces/` 一起备份，确保账号、余额、流水、素材和任务状态保持同一时间点。

### 9.3 恢复

1. 停止服务。
2. 恢复同一批次的 `system/` 和 `workspaces/`。
3. 检查目录所有者和读写权限。
4. 启动服务。
5. 管理员登录后核对计费规则、团队余额和最近流水。

不要只恢复 `billing-accounts.json` 而丢失 `billing-ledger.jsonl`，否则余额仍正确，但历史流水不完整。

## 10. 验证清单

上线前至少验证：

1. 计费关闭时，余额为 0 仍可生图和分析。
2. 开启计费后，余额不足会在上游调用前失败。
3. 成功生图一次只扣一次；接口自动重试成功仍只扣一次。
4. 生图全部失败不扣费。
5. 模板分析成功扣一次；失败三次不扣费。
6. AI 质检触发复核时，两个成功 LLM 调用分别计费。
7. 直接复制、跳过、不处理不扣费。
8. 多任务并发时不会花超余额。
9. 普通成员访问管理员计费接口返回 403。
10. 管理员充值、扣减、修改规则后，左侧余额和流水能及时更新。
