# 淘宝发布辅助第一版设计

## 目标

在现有 Web 项目中增加淘宝发布辅助能力。人工筛图完成并整套通过后，运营可以在 Web 端选择固定淘宝类目模板，点击“发布到淘宝草稿”。浏览器插件收到任务后，在已登录的淘宝卖家后台自动填写商品发布页面，上传图片，并保存为草稿。

第一版只保存草稿，不点击立即上架。

## 不在范围内

- 不修改套餐、生图、提示词、计费逻辑。
- 不做淘宝正式上架点击。
- 不做所有淘宝类目的通用发布器。
- 不做服务器部署发布辅助。
- 不绕过淘宝登录、风控、验证码或人工校验。

## 固定类目模板

第一版内置以下类目模板：

- 餐边柜（储物柜）
- 边角柜
- 电视柜
- 床头柜
- 衣柜
- 鞋柜
- 屏风柜
- 斗柜
- 茶几
- 梳妆台
- 衣帽架
- 书柜
- 异形产品
- 多巴胺 包豪斯

每个模板保存一组淘宝发布预设字段：

- 淘宝发布链接
- 默认商品标题前缀或标题规则
- 默认价格
- 默认库存
- 发货地
- 运费模板
- 售后/服务模板
- 商品属性默认值
- 主图、3:4 主图、详情图的图片匹配规则
- 保存草稿按钮识别规则

字段允许先为空。后续通过实际淘宝后台观察逐步补齐。

## Web 端页面

新增“淘宝发布辅助”页面。

页面只同步人工筛图中满足以下条件的任务：

- 任务已进入人工筛图结果列表。
- 任务内所有需要处理的图片都已人工通过。
- 任务不是生成中、缺图、失败或待确认状态。

任务卡片展示：

- 任务名
- 图片数量
- 标题状态
- 当前发布类目
- 发布状态
- 最近失败原因
- 操作按钮：选择类目、检查任务包、发布到淘宝草稿、重置状态

发布状态：

- 未配置
- 待发布
- 等待插件接收
- 插件已接收
- 正在打开淘宝页面
- 正在填写字段
- 正在上传图片
- 正在保存草稿
- 已保存草稿
- 失败

## 发布任务包

Web 端在点击“发布到淘宝草稿”时生成任务包。

任务包包含：

- 发布任务 ID
- 任务文件夹路径
- 任务名
- 类目模板 ID
- 类目模板字段
- 商品标题
- 主图列表
- 3:4 主图列表
- 详情图列表
- 所有图片的 Web 可访问 URL
- 文件名和相对路径
- 创建时间

标题来源：

1. 优先读取任务目录中的 `标题.xlsx` 第一条标题。
2. 如果没有标题，Web 端阻止发布，并提示先生成标题。

图片来源：

1. 读取人工筛图已通过图片的生成结果。
2. 按类目模板中的图片规则分类为主图、3:4 主图、详情图。
3. 无法自动分类时，Web 端允许人工调整图片用途。

插件不直接读取本地磁盘。图片通过 Web 端文件接口提供 URL，插件下载后上传到淘宝页面。

## Web 端与插件通信

第一版采用 Web 端主动发布、插件自动接收。

通信方式：

1. Web 端提供本地 HTTP 接口管理发布任务。
2. 浏览器插件后台脚本定时轮询或保持轻量连接，发现待接收任务后自动领取。
3. 插件领取后将任务状态更新为“插件已接收”。
4. 插件每个阶段回传状态和错误信息。

接口草案：

- `GET /api/taobao/publish/tasks`：列出 Web 端发布辅助任务。
- `POST /api/taobao/publish/tasks/:id/queue`：将任务入队，等待插件接收。
- `POST /api/taobao/publish/claim`：插件领取下一个等待接收的任务。
- `POST /api/taobao/publish/tasks/:id/status`：插件回传状态。
- `GET /api/taobao/publish/tasks/:id/package`：插件读取任务包。

插件需要一个本地连接令牌。令牌由 Web 端发布辅助页显示，运营首次安装插件后填入插件配置，用来避免任意网页调用本地发布接口。

## 浏览器插件

插件目录建议放在 `extensions/taobao-publish-assistant`。

插件包含：

- `manifest.json`
- 后台脚本：连接 Web 端、领取任务、管理状态。
- content script：在淘宝发布页执行填表、上传、保存草稿。
- popup 页面：显示连接状态、当前任务、最近错误、手动重试按钮。

插件权限：

- 访问本地 Web 端地址，例如 `http://127.0.0.1:*/*`。
- 访问淘宝卖家后台发布页面。
- tabs、scripting、storage。

## 淘宝页面执行策略

淘宝后台页面可能不是简单 DOM。插件执行必须支持多策略：

1. 优先使用稳定 DOM 选择器。
2. 选择器失败时，根据文本标签定位相邻输入框或按钮。
3. 图片上传优先查找真实 `input[type=file]`。
4. 找不到文件输入时，模拟点击上传区域，再根据浏览器能力尝试文件注入、拖拽或剪贴板方案。
5. 保存草稿成功必须通过页面提示、按钮状态、URL 变化或草稿标识确认。

第一版需要实际登录淘宝后台后进行页面观察，记录：

- 每个固定类目的发布入口 URL。
- 必填字段列表。
- 默认可预设字段。
- 主图上传控件。
- 3:4 主图上传控件。
- 详情页图片编辑区域。
- 保存草稿按钮。
- 保存成功提示。

观察结果会反写到类目模板和插件页面适配器中。

## 错误处理

插件遇到以下情况应停止并回传失败原因：

- 未登录淘宝。
- 页面不是商品发布页。
- 类目模板缺少发布链接。
- 标题缺失。
- 主图或详情图缺失。
- 必填字段找不到。
- 图片上传失败。
- 保存草稿按钮找不到。
- 淘宝弹出验证码、二次确认或风控页面。

失败任务可在 Web 端重置为待发布。

## 验收标准

第一版完成时应满足：

1. Web 端存在淘宝发布辅助页面。
2. 页面能同步所有人工筛图整套通过的任务。
3. 页面内置 14 个固定类目模板。
4. 任务可以选择类目并生成发布任务包。
5. 点击“发布到淘宝草稿”后，任务进入等待插件接收状态。
6. 浏览器插件能连接 Web 端并自动接收任务。
7. 插件能在淘宝后台读取任务包并尝试填写发布页。
8. 插件能回传执行状态和失败原因。
9. 经过淘宝后台实测后，至少一个类目能完整填写必填项、上传图片并保存草稿。
10. 构建和现有测试通过，且套餐、生图、计费相关测试不回退。
